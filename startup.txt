=======================================
  VEND88 Dashboard - Startup Commands
=======================================

--- FIRST TIME SETUP (run once) ---

1. Install frontend dependencies:
   npm install

2. Create backend virtual environment:
   cd backend
   python -m venv venv

3. Install backend dependencies:
   cd backend
   venv\Scripts\activate
   pip install -r requirements.txt

--- RUN THE APP (every time) ---

Terminal 1 - Backend:
   cd backend
   venv\Scripts\activate
   uvicorn main:app --reload

Terminal 2 - Frontend:
   npm start
   # API target switch (optional):
   # $env:REACT_APP_API_TARGET="official"
   # $env:REACT_APP_API_TARGET="custom"
.
--- URLs ---
Frontend:  http://localhost:3000
Backend:   http://localhost:8000
API Docs:  http://localhost:8000/docs

--- RUN MOBILE APP WITH EXPO GO ---

Terminal 3 - Mobile:
   cd mobile
   npm install
   npx expo start --lan

On iPhone (Expo Go):
   1. Install Expo Go from App Store
   2. Scan the QR code from terminal

On Android (Expo Go):
   1. Install Expo Go from Play Store
   2. Scan the QR code from terminal

--- MOBILE API CONFIG ---

1. Choose API target:
   $env:EXPO_PUBLIC_API_TARGET="official"
   # or
   $env:EXPO_PUBLIC_API_TARGET="custom"
2. If using custom backend, set custom API base URL:
   $env:EXPO_PUBLIC_CUSTOM_API_BASE_URL="http://192.168.1.55:8000/api/v1"
3. Start backend with host binding:
   uvicorn main:app --reload --host 0.0.0.0 --port 8000
4. In mobile terminal, start Expo:
   npx expo start --lan

--- EAS COMMANDS ---

From the mobile folder:
   cd mobile
   npx --yes eas-cli login

Important:
   - Run EAS commands from the mobile folder only, or use the root wrapper scripts.
   - Running `eas build` from the workspace root can pick up the root `app.json` and cause Expo project slug/projectId mismatch errors.

Preflight before using EAS build quota:
   node -v
   java -version
   npx expo-doctor
   .\android\gradlew.bat :app:assembleDebug --stacktrace
   .\android\gradlew.bat :app:assembleRelease --stacktrace

Requirements for local Android preflight:
   - Use Node 20.19.4 or newer.
   - Install JDK 17 and set JAVA_HOME before running gradlew.
   - If local Gradle assemble passes, the EAS gradlew phase is much less likely to fail on native compile issues.

Run in Expo Go:
   npx expo start

Push an OTA update:
   npx --yes eas-cli update --branch production --message "update message"

Create a cloud build:
   npx --yes eas-cli build
   # or from repo root:
   npm run eas:preview

Submit a build to the stores:
   npx --yes eas-cli submit

Note:
   - localhost does not work from phone unless using tunnel/proxy mode.
   - LAN mode is recommended first for stability and speed.
   - If LAN cannot connect, use fallback:
     npx expo start --tunnel
   - Login currently supports demo mode if backend auth endpoint is not available.
